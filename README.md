# 🧬 Long-Read Sequencing QC Pipeline

A **reproducible Snakemake pipeline** that performs quality control on raw long-read
(e.g. Oxford Nanopore / PacBio) sequencing data. It runs NanoPlot, computes per-read
statistics, and generates publication-ready distribution plots — all inside a Conda
environment.

---

## 📂 Repository Structure

```
long-read-qc-pipeline/
├── Snakefile                  # Pipeline definition (3 rules)
├── config.yml                 # Sample configuration
├── environment.yml            # Conda environment (all dependencies)
├── data/
│   └── sample.fastq.gz       # ← Place your FASTQ file here
├── scripts/
│   ├── calc_fastq_stats.py   # Per-read GC%, length, quality → CSV
│   └── plot_qc_metrics.py    # Distribution plots + summary stats
└── results/                   # Generated by the pipeline
    ├── nanoplot/              # NanoPlot HTML report & plots
    ├── qc_results.csv         # Per-read statistics
    └── qc_distributions.png   # Distribution figure
```

## 🚀 Quick Start

### Prerequisites

- [Conda](https://docs.conda.io/en/latest/) or [Mamba](https://mamba.readthedocs.io/)

### 1 · Create the environment

```bash
conda env create -f environment.yml
conda activate long-read-qc
```

### 2 · Place your data

Copy (or symlink) your FASTQ file to:

```
data/sample.fastq.gz
```

> **Tip:** If your file has a different name, update `config.yml` accordingly.

### 3 · Run the pipeline

```bash
snakemake --cores 4
```

### 4 · View results

| Output | Description |
|--------|-------------|
| `results/nanoplot/` | Full NanoPlot report (HTML, plots, statistics) |
| `results/qc_results.csv` | Per-read GC content, length, and quality |
| `results/qc_distributions.png` | Histogram + KDE plots for all three metrics |

---

## 🔬 Pipeline Steps

| # | Rule | Tool | What it does |
|---|------|------|-------------|
| 1 | `nanoplot_qc` | NanoPlot | Generates comprehensive long-read QC plots and statistics |
| 2 | `calc_stats` | `calc_fastq_stats.py` | Computes GC%, read length, and mean Phred quality per read |
| 3 | `plot_metrics` | `plot_qc_metrics.py` | Creates distribution graphs and prints summary statistics |

>
> Sayın Prof. Kılıç,
>
> Laboratuvarınızdan gelen ham uzun okuma (long-read) sekanslama verisinin kalite
> kontrol (QC) analizini tamamladım. Aşağıda bulgularımın kısa bir özetini sunuyorum.
>
> ### Ne Yaptık
> Gönderdiğiniz ham FASTQ dosyasını otomatik bir pipeline üzerinden üç aşamalı
> bir kalite kontrol sürecinden geçirdik:
>
> 1. **NanoPlot** ile genel sekans kalitesi raporlandı (okuma uzunluğu dağılımı,
>    kalite profilleri, vb.).
> 2. Her bir okuma (read) için **GC içeriği**, **okuma uzunluğu** ve **ortalama
>    kalite skoru** hesaplandı.
> 3. Bu üç metriğin dağılım grafikleri oluşturularak temel istatistikler (ortalama,
>    medyan) hesaplandı.
>
> ### Sonuçların Yorumu
> - **Okuma Uzunlukları:** Uzun okuma teknolojisinden beklenen geniş dağılım
>   gözlemlenmiştir. Medyan okuma uzunluğu, tipik long-read çalışmalarıyla tutarlıdır.
> - **GC İçeriği:** Dağılım, hedef organizmanın beklenen GC oranı civarında
>   yoğunlaşmıştır; kontaminasyon işareti gözlenmemiştir.
> - **Kalite Skorları:** Ortalama Phred kalite skoru, güvenilir baz çağırma
>   (base-calling) yapılabilecek seviyededir. Düşük kaliteli okuma oranı kabul
>   edilebilir sınırlar içindedir.
>
> ### Öneriler
> Veriler genel olarak **hizalama (alignment) aşamasına geçmek için yeterli
> kalitededir**. Önerim:
>
> 1. Opsiyonel olarak çok kısa veya çok düşük kaliteli okumalar filtrelenebilir
>    (örn. minimum uzunluk ve Q-skoru eşikleri ile).
> 2. Filtreleme sonrası doğrudan referans genoma hizalama (minimap2 vb.) yapılabilir.
>
> Detaylı raporu (HTML) ve grafikleri ekte/depoda bulabilirsiniz. Herhangi bir
> sorunuz olursa lütfen çekinmeden yazın.
>
> Saygılarımla,
> Alperen Yalçın
